Python Web Services

Welcome to pywebsvcs, a SourceForge "meta-project" for Python web services programming. It supports a range of libraries for both client and server programming, using protocols like SOAP, WSDL, UDDI, etc. The Python home page is the homepage for the Python programming language. Web services are built on XML. For information on Python and XML, check out the Python XML SIG.

Projects

pywebsvcs currently features the following subprojects:

SOAPpy

A SOAP implementation written by Cayce Ullman and Brian Matthews, now maintained by Gregory Warnes and Christopher Blunck.

API docs | README file | CVS tree | Latest official release (0.11.0)

ZSI, Zolera SOAP Infrastructure

A different SOAP implementation initially by Rich Salz, now maintained mainly by Rich and Christopher. The wsdl2python tools are by LBNL.

API docs | Documentation | CVS tree | Latest official release (1.4.1)

wstools

A library of WSDL, and other, classes useful for building Web Services stacks. This is common code used by SOAPpy and ZSI, and bundled with it.

API docs | CVS tree | No separate release

The API docs are updated daily from the CVS repository.

Other information

The SF Project Page has links to all the project management information (bugs, releases, etc). There are often beta or release candidates available; the links above give only the last official release.

The SOAPy project at SF includes parsers for WSDL and Schema.
